SENDMD5 RECVMD5
SENDTHROUGH
RECVTHROUGH
SENDMD5THROUGH
RECVMD5THROUGH

Le mode utilise pour un transfert avec cette regle. Par exemple, SENDM5 conduira a un transfert de fichier depuis le requeteur vers le requete en utilisant pour chaque paquet un test de type MD5.

Si aucun Host ne sont ecrits, alors la regle autorise tous les hosts. Si des hosts sont specifies, seuls ceux-ci seront autorises avec cette regle.

Le format est comme suit:

<hostids> <hostid>hostid1</hostid> <hostid>hostid2</hostid> </hostids>

Si non null, specifie le chemin pour la reception.

Si non null, specifie le chemin pour l'emission.

Si non null, specifie le chemin pour les archives (export).

Si non null, specifie le chemin pour l'espace de travail (durant la reception).

Recp Pre Recp Post Recp Error Emiss Pre Emiss Post Emiss Error  

Specifie les actions a realiser AVANT le transfert depuis le recepteur (SEND et requete, ou RECV et requeteur).

Le format est comme suit:

<tasks> <task> <type>NAME</type> <path>path</path> <delay>x</delay> </task> </tasks>ou
-type est un parmis LOG, MOVE, MOVERENAME, COPY, COPYRENAME, EXEC, EXECMOVE, EXECOUTPUT, EXECJAVA, TRANSFER, VALIDFILEPATH, DELETE, LINKRENAME, TAR, ZIP, TRANSCODE, RENAME, FTP
-path est un argument statique ou des remplacements pour se faire
-delay est usuellement le delai maximale pour l'execution
Voir l'API ou la documentation pour les conditions specifiques.
